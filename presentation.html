🔥 The Problem

⏱️ Slow Triage

IT teams manually read every ticket before even starting to solve it. High-priority issues wait in the same queue as low-priority ones.

📋 Repetitive Issues

VPN disconnects, password resets, Outlook sync failures — the same 10 issues account for 80% of tickets. Yet agents answer from scratch each time.

🚨 Missed SLAs

Without automated tracking, high-priority tickets silently breach SLA while agents handle whatever's on top of the pile.

💡 Our Solution

🤖 AI Agent

Claude Sonnet 4.6 via AWS Bedrock reads each ticket, finds the relevant knowledge base article, and returns a structured step-by-step solution — no hallucination, no invented infrastructure.

⚡ Instant Escalation

SLA rules fire before the API call. If the SLA is already breached, the ticket is auto-escalated with a reason — zero tokens spent.

🗂️ Multi-source Ingestion

Reads from simulated Slack, Teams, Email, and Service Desk sources in one unified pipeline. Extensible to real APIs.

📊 Streamlit Dashboard

Live ticket board with SLA countdowns, status colours, one-click agent run, and backend toggle (Bedrock / direct API).

🤖 AI Agent — Bedrock Integration

Dual Backend

Single env var switches between AWS Bedrock and direct Anthropic API:

# AWS Bedrock (default)
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_PROFILE=bootcamp
export AWS_REGION=us-west-2

# Direct Anthropic API (fallback)
export ANTHROPIC_API_KEY=sk-ant-...

Prompt Caching

System prompt is sent with cache_control: ephemeral — cached across all tickets in one run, reducing latency and token cost.
